## 一、系统简介

本系统是由 AI（Claude）驱动的全自动交易机器人，支持以下功能：

| 功能 | 说明 |
|------|------|
| **AI 市场分析** | 自动判断当前是震荡行情还是趋势行情 |
| **双策略切换** | 震荡行情：低买高卖；趋势行情：顺势跟单 |
| **自动止损止盈** | 固定止损 + 固定止盈 + 移动止损三重保护 |
| **实时价格监控** | 每隔数秒检查持仓，及时平仓 |
| **多通道通知** | 微信 / 飞书 / QQ / Telegram 实时推送 |
| **多平台支持** | 币安 / OKX / Polymarket 同时运行 |

**运行方式：** 本系统安装在您的本地电脑，24 小时在后台自动运行，无需人工干预。

## 二、安装步骤

### 前提条件
- 电脑系统：macOS 或 Windows（需 WSL）或 Linux
- 已安装 Python 3.9 或更高版本

### 一键安装（服务商协助完成）

```bash
# 在项目文件夹中执行
bash setup.sh
```

安装程序会自动完成：
1. 检测系统环境
2. 安装所需依赖库
3. 安装 Claude Code
4. 引导填写 API Key
5. 验证连接是否成功
6. 可选：设置开机自启动

> **注：** 首次安装由服务商工作人员协助完成，全程约 10 分钟。

## 三、填写配置文件

配置文件位于：`config/user_config.yaml`

可用文本编辑器（如 VSCode、记事本）打开修改。

### 3.1 必填项

```yaml
exchanges:
  binance:
    enabled: true          # 改为 true 启用币安
    api_key: "填入你的KEY"
    api_secret: "填入你的SECRET"
```

### 3.2 设置交易资金

```yaml
risk:
  total_capital: 1000      # 填入你的交易总资金（USDT）
```

### 3.3 选择交易对

```yaml
markets:
  binance:
    symbols:
      - "BTC/USDT"         # 比特币
      - "ETH/USDT"         # 以太坊
      - "SOL/USDT"         # 可自行增减
```

## 四、获取 API Key

### 4.1 币安（Binance）

1. 登录 [binance.com](https://www.binance.com)
2. 点击右上角头像 → **API 管理**
3. 点击 **创建 API**，选择"系统生成"
4. 填写备注（如：交易机器人）
5. 完成安全验证
6. **权限设置**：
   - ✅ 启用现货和杠杆交易
   - ✅ 允许读取
   - ❌ 不要开启提币权限（安全起见）
7. 复制 **API Key** 和 **Secret Key** 填入配置文件

> ⚠️ **安全提示：** API Key 只显示一次，请立即保存。建议绑定 IP 白名单（填入你电脑的公网 IP）。

---

### 4.2 OKX

1. 登录 [okx.com](https://www.okx.com)
2. 右上角头像 → **API**
3. 点击 **创建 API Key**
4. 填写名称，设置密码短语（Passphrase）
5. **权限选择**：交易 ✅，读取 ✅，提币 ❌
6. 复制 API Key、Secret Key、Passphrase 填入配置文件

---

### 4.3 Polymarket

1. 安装 [MetaMask](https://metamask.io) 钱包浏览器插件
2. 创建或导入钱包，切换到 **Polygon 网络**
3. 向钱包转入 USDC（Polygon 链）
4. 在 MetaMask 中：设置 → 账户详情 → 导出私钥
5. 将私钥填入配置文件

> ⚠️ **安全提示：** 私钥是资金的唯一凭证，绝对不要泄露给任何人，包括服务商。

## 五、配置通知

在 `config/user_config.yaml` 的 `notifications` 部分配置，**至少开启一个通道**。

### 5.1 微信通知（推荐 · 最简单）

**使用 PushPlus：**

1. 手机微信扫码访问 [pushplus.plus](https://www.pushplus.plus)
2. 关注公众号并登录
3. 首页复制你的 **Token**
4. 填入配置文件：

```yaml
wechat_pushplus:
  enabled: true
  token: "粘贴你的token"
```

5. 此后交易信号会直接推送到你的微信

---

### 5.2 飞书通知

1. 打开飞书，进入你的群组
2. 右上角 **设置** → **机器人** → **添加机器人**
3. 选择 **自定义机器人** → 填写名称
4. 复制 **Webhook URL**
5. 填入配置文件：

```yaml
feishu:
  enabled: true
  webhook_url: "粘贴webhook地址"
```

---

### 5.3 QQ 通知

QQ 通知需要在本地运行 go-cqhttp 程序（较复杂，建议由服务商协助配置）：

```yaml
qq:
  enabled: true
  host: "http://127.0.0.1:5700"
  user_id: "接收消息的QQ号"
```

---

### 5.4 Telegram 通知

1. 在 Telegram 中搜索 **@BotFather**
2. 发送 `/newbot`，按提示创建机器人，获取 **Token**
3. 搜索 **@userinfobot** 获取你的 **Chat ID**
4. 填入配置文件：

```yaml
telegram:
  enabled: true
  bot_token: "你的bot token"
  chat_id: "你的chat id"
```

## 六、启动交易

### 6.1 首次启动前 · 验证配置

```bash
cd trading-skill
python verify.py
```

所有项目显示 ✓ 后再启动实盘。

### 6.2 模拟交易测试（强烈建议先跑一次）

```bash
python test_paper_trading.py
```

模拟交易使用真实行情，但不产生真实订单，用于验证策略逻辑。

### 6.3 启动实盘交易

```bash
# 激活虚拟环境（每次重启终端需执行）
source .venv/bin/activate

# 启动
python main.py
```

启动后系统会：
- 每 **1 小时** 分析一次市场，寻找交易机会
- 每 **N 秒**（你设置的 monitor_interval）检查一次持仓，触发止损止盈
- 有信号时推送通知到你配置的通道

> 启动后可最小化终端窗口，系统在后台持续运行。不要关闭终端。

### 6.4 后台持续运行（推荐）

```bash
# 使用 nohup 让程序在关闭终端后继续运行
nohup python main.py > logs/stdout.log 2>&1 &

# 查看运行状态
tail -f logs/stdout.log
```

## 七、常用命令

在项目目录下执行：

```bash
# 查看当前持仓和今日盈亏
python main.py status

# 平掉所有持仓
python main.py close all

# 平掉指定交易对
python main.py close BTC/USDT

# 查看收益和返佣报告
python main.py report

# 暂停交易（修改配置文件）
# 打开 config/user_config.yaml，将 system.enabled 改为 false
```

在 **Claude Code** 中也可以使用 `/trading` 命令：

```
/trading              → 立即运行一次分析和交易
/trading status       → 查看持仓状态
/trading close all    → 平仓所有
/trading report       → 查看报告
/trading pause        → 暂停
/trading resume       → 恢复
```

## 八、策略说明

### 8.1 AI 如何判断行情

系统每小时自动计算以下指标：

| 指标 | 作用 |
|------|------|
| **ADX** | 趋势强度，>25 判断为趋势行情 |
| **RSI** | 超买超卖，<35 超卖，>65 超买 |
| **布林带** | 价格波动区间，判断是否处于震荡 |
| **MACD** | 动量方向，金叉/死叉确认趋势 |
| **均线** | MA20/MA60 排列判断多空方向 |
| **成交量** | 放量确认信号有效性 |

### 8.2 震荡策略（横盘行情）

**触发条件：** ADX < 20，价格在布林带内震荡

**操作逻辑：**
- 价格跌到布林下轨 + RSI < 35 → **买入**（做多）
- 价格涨到布林上轨 + RSI > 65 → **卖出**（做空）
- 价格回归中轨时 → **自动平仓**

**适用场景：** BTC、ETH 等主流币的横盘整理阶段

### 8.3 趋势策略（单边行情）

**触发条件：** ADX > 25，MA20 明显偏离 MA60

**做多操作：**
- MA20 上穿 MA60（金叉）+ MACD 柱翻正 + 成交量放大 → **买入**
- 移动止损跟随价格上涨，价格回撤触及移动止损 → **自动平仓**

**做空操作：**
- MA20 下穿 MA60（死叉）+ MACD 柱翻负 + 成交量放大 → **做空**
- 同上，移动止损跟随价格下跌

**适用场景：** 牛市拉升、熊市下跌等单边趋势行情

## 九、风险参数说明

在 `config/user_config.yaml` 的 `risk` 部分调整：

```yaml
risk:
  total_capital: 1000       # 投入总资金
  max_positions: 3          # 最多同时开几个仓位
  position_size_pct: 0.10   # 每笔用总资金的10%
  stop_loss_pct: 0.015      # 止损 1.5%（到达后强制平仓）
  take_profit_pct: 0.03     # 止盈 3.0%（到达后自动获利了结）
  max_daily_loss: 0.05      # 当日亏损超过总资金5%，停止当天交易
  risk_reward_ratio: 2.0    # 盈亏比低于2:1的信号不执行
```

### 参数建议

| 风险偏好 | 止损 | 止盈 | 单笔仓位 | 当日止损 |
|---------|------|------|---------|---------|
| 保守型 | 1.0% | 2.0% | 5% | 3% |
| 稳健型（默认） | 1.5% | 3.0% | 10% | 5% |
| 激进型 | 2.0% | 5.0% | 15% | 8% |

> ⚠️ **重要提示：** 建议从保守型参数开始，熟悉系统后再调整。

## 十、常见问题

**Q：系统显示"暂无交易信号"是正常的吗？**

A：完全正常。系统有严格的入场条件，宁可错过也不乱入场。等待合适的机会是正确的策略行为。

---

**Q：如何知道系统还在运行？**

A：
1. 检查通知——每天会发一次日报到你的微信/飞书
2. 查看日志文件：`logs/system.log`
3. 执行 `python main.py status` 查看当前状态

---

**Q：API Key 安全吗？**

A：API Key 只存储在你本地电脑的配置文件中，不会上传到任何服务器。建议：
- 不开启提币权限
- 绑定 IP 白名单（只允许你的电脑访问）
- 定期更换 API Key

---

**Q：程序崩溃或电脑重启后怎么办？**

A：如果安装时选择了开机自启，系统会自动重启。否则需要手动执行：
```bash
cd trading-skill && source .venv/bin/activate && python main.py
```

---

**Q：如何彻底停止所有交易？**

A：两种方式：
1. 快速停止：终端按 `Ctrl+C`，然后执行 `python main.py close all` 平掉所有仓位
2. 配置文件关闭：将 `system.enabled` 改为 `false`，下次启动不会开仓

---

**Q：收益如何结算？**

A：系统按月自动统计盈利，执行 `python main.py report` 可查看详细报告，其中包含：
- 当月总盈利
- 应付利润分成金额
- 历史交易记录

---

## 十一、注意事项

1. **不要在配置文件中开启提币权限**，交易机器人只需要交易权限即可

2. **建议先用小资金测试**（如 100-200 USDT），跑 1-2 周确认策略表现后再加仓

3. **不要频繁修改策略参数**，给系统足够的时间在不同行情中表现

4. **止损是保护你的资金的**，不要把止损设得太宽，单笔亏损控制在总资金 2% 以内

5. **本系统不保证盈利**，量化交易存在亏损风险，请只投入你能承受亏损的资金

6. **保持网络稳定**，交易系统需要持续联网，断网可能导致无法及时平仓

7. **定期查看日志**，关注 `logs/system.log` 中是否有异常报错
